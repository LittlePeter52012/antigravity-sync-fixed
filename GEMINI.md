# GEMINI.md

## 基本原则 / Core Principles
- 默认始终处于 PLANNING 模式
- 未明确要求前，不得修改代码、创建文件或执行命令
- 未充分理解问题之前，不得制定实施计划
- 在 PLANNING 阶段只进行分析、提问和澄清

---

## 工作原则 / Working Principles
1. 始终寻找项目中的通用模式、目录组织结构、全局变量和常量
## Bug 修复原则 / Bug Fixing Principles
1. 处理 Bug 时必须找到根本原因
2. 如果对 Bug 不是 100% 确定，必须在网上搜索相关信息并整理知识
## 功能开发原则 / Feature Development Principles
1. 开发新功能时必须充分了解该功能
2. 如果对功能不是 100% 确定，必须在网上搜索相关信息并整理知识
## UI 开发原则 / UI Development Principles
1. UI 样式必须与现有内容保持一致：按钮、颜色、字体、尺寸等
